HTML Comments and Why You Should Use Them

Most coding languages have some way of adding comment to your code. Comments are an extreemly important part of any coding language. To make comments in HTML we use the comment tag <!-- to open a comment, and -->. When put together it looks like this <!--comment-->. A comment tag like this allows you to place comments directly into your HTML source code because the tag tells your browser to ignore the writing inside and not display it to the screen. Comments can be used for anything, but code comments are particularly useful because they allow us as programers to describe sections of code, leave notes for ourselves, and describe why things have been coded how they were. Using comments for some of the ways mention above can improve clarity and mantainability by help make your code clearer and understandable to others, and to yourself down the line. One tip worth noting is that in VSCode you can select any line then use use Ctrl + / to turn the link into a comment or a blank comment with an empty line. Using the shortcut again uncomments the line. For more information on developing good commenting practices: check out this article .

Why Some HTML Tags and Attributes Become Deprecated

The term "deprecated" is a common piece of code termonology used with many diffrent languages. The term deprecated is used to indicate that the use of a piece of code is being discourged as it's phased out. Often, when something is deprecated this means that it's being replaced by a diffrent standard or technique for acomplishing the task that might be preferable to the former. We choose to deprecat things rather than just removing them because this gives developers time to transition away and start using alternative stratiges and aviod breaking code.

With HTML, we see tags like the font tag <font>, the centering tag <center>, the text size tag <big>, the strikethrough tag <strike>, and a number of other tags are all concidered deprecated. We also attributes like "align", "size", "text" (for text color), and others have been deprcated for some uses. So why are these tags deprecated? Well, in this case, many though not all of the deprecated tags and attributes have been deprecated because HTML is moving away from styling/formating the page using any HTML. HTML should strictly be used for adding the content to the page, for styling, CSS should be used. The use of CSS over HTML for styling has a number of important benifits. CSS styling allows for standardization and consitency cross platform, cleaner code, and it makes changing styling across your page (internal CSS), or even across your entire site extreemly simple (external CSS).
